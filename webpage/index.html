FMake
By Filipe Chagas

FMake is a Python script designed to automate the process of compiling programs with many modules. It can be used as an alternative to GNU Make or CMake.

FMake is free software. Use, reproduce and adapt it to your needs without restriction.

Introduction

FMake is simple and robust, there is nothing very special about it. It consists of the fmake directory script set.

Basically, the fmake/main.py script has some trace functions for code files. In addition, the fmake/targets directory contains a set of plug-ins responsible for different compilation modes.

The compilation process is described simply and clearly in a text file containing fmake settings. This file is not exactly a script, but it is a Python dictionary that will be parsed by an eval().

How to use

1 - Add the fmake directory to the project root

This directory contains the fmake scripts.

2 - Create a FMake config text file in the project root

The contents of this file must be a python dictionary with the compilation process information. This file will be further explained in the documentation.

3 - Create a build.py script in the project root

This script file must call the make function from the fmake/main.py module (specifying the project root path, fmake directory path, and settings file path).

4 - Are native targets enough?

You may need to create your own target script.

Operation

The fmake/main.py script contains the make function. This function must be called by a build.py script to start the build process.

One of the first things that happens when make is invoked is the loading of target-modules into the target directory. This operation is performed by the load_targets function.

Immediately afterwards, the configurations file is loaded and parsed by the eval function.

If there are no syntax errors in the configurations file, the step-by-step compilation will start.

Configurations file

The configurations file is a text file that contains a python dictionary with the compilation process information.

The configuration file must contain:

  • A "magic" tag with the value "fmakeconfig".
  • A "steps" tag with a list containing the tags of the compilation steps in the order in which they should be performed.
  • Arbitrary tags (can be strings or numbers) containing the information of the compilation steps. Each must contain a sub-dictionary with the build target settings.

Visit the operation README for more informations

Targets and target-modules

Target-modules are small plug-ins that perform the most important part of the compilation process. For each build step described in the fmake configuration file, a target is specified and a target-module is used.
